交并补

发表于 2019-08-13 | 分类于 基础算法

思路

  1. 将集合的元素用map进行存储，key为集合的元素，value为1.
  2. 对于交集遍历其中一个集合（选择长度较小的好些），如果另一个集合存在这个元素，则该元素是交集中的元素。
  3. 对于并集，先取一个集合B作为并集的结果，对A做遍历，如果B中不存在该元素，则该元素是并集中的元素。
  4. 对于补集，如求A的补集，对B做遍历，如果A中不存在该元素，则该元素是补集中的元素。
