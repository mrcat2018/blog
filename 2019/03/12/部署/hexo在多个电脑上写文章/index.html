hexo在多个电脑上写文章

发表于 2019-03-12 | 分类于 部署

        之前用hexo试了试水,当时感觉hexo写文章并部署到github挺方便的，但是没啥写的就放下了。半年后感觉要开始写点东西了。准备写时就发现了个问题，怎么在多个设备之间同步。搜了很多教程，觉得这个教程 还可以，就是说的有点简单了。

前提条件

必须有hexo的全部工程文件，如果只有之前上传在github的文章，那只能新建hexo项目了。

原理

git 管理两个分支，一个master分支，一个src分支。master分支由src分支通过hexo g -d 生成并上传到远端。src分支保留工程必须的文件，以便在其他电脑clone后可以直接写文章。

开始

  1. 在blog目录下建立一个src分支

    git checkout -b src
    git branch 可以查看当前的分支

  2. 添加全部

    git add .

  3. 提交到本地

    git commmit -m “新建源分支”

  4. 推送到github (这一步可能会要求输入github账号和密码)

    git push origin src

怎么使用呢。

在另一个设备上克隆src分支
git clone -b src https://github.com/XXX/blog.git
因为之前上传的项目是没有模块的，此时还需要安装hexo 模块
npm install(我嫌npm太慢，用的淘宝的cnpm install)
之后就可以在source/_post下写文章了。
发布还是 hexo g -d
写完后要将源文件上传
在blog目录下
git add .
git commit -m “新增了篇文章”
git push origin src

大功告成

可能遇到的问题

  1. WARN No layout: index.html 这是由于主题文件缺失造成的，有两种方法解决。

    1. 直接拷贝一份相关主题文件到项目的theme目录下面

    这样做，每次在新的设备上都得拷贝一份，如果修改了主题的样式，还不会同步到github上，只会同步到文章发布的master分支上。不推荐

    1. 自己先fork一份主题文件到自己的远端仓库，然后给blog项目添加子模块。怎么fork就不说了。

    这样修改样式后可以提交到远端仓库，保证每次clone时都是自己（可能做过修改）的主题

    clone 时的加上 –recursive

  2. fatal: in unpopulated submodule ‘.deploy_git’

    一般执行过npm install 都会直接在项目的依赖包中安装了hexo-deployer-git的，先考虑是不是git缓存不一致造成的。
